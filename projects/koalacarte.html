KoÀlacarte: App to make informed food choices

Developed application as a final project for Information Organization Lab in a team of three with Nico Soldi and Ganesh Viswanathan. Full report

Problem:

If you experience an uncertainty about the food you are about to order at a restaurant, specifically if it could contain a serious food allergen, then you are one among approximately 15 million Americans affected by food allergy. KoÁlacarte is a Service that allows user to identify dishes that are safe to eat in a restaurant keeping in mind dietary restrictions and preferences.

Process:

The first step in the process was collecting a database of menus. Next step was to match dietary preference of users at ingredient level to menu item. This was done by collecting recipes for all the items in the menu. Multiple recipes were collected for single dish as different recipes differed slightly in the ingredients they use. This approach was used to be give user a probability score of an ingredient occurring in a menu item.

My role in this project was limited to application development. We decided that the use case of the app is mostly in a restaurant setting so the app must be available on the go. Therefore we decided to implement the app as a mobile app rather than a desktop app.

We build the app using Ionic framework. We made the choice as Ionic can be used to make hybrid mobile apps easily. Code for Iocnic is written in HTML5, CSS and AnglarJS, Ionic creates native code for Android and iOS using the underlying Cordova API. It Works like a single page application with one index.html and different templates plugin as needed. JS part divided into route, controller, services, directives and app, like a typical application in angular.

We built a database model to include two main entities, the user and the restaurant. The user could store the ingredient that they have to restrict in their diet in the database through the onboarding process. The restaurant has a menu, each menu has a list of dishes and dishes had a list of ingredients in them.

The application uses hybrid database to manage information and an API layer to delegate information retrieval and storage in the database. The primary ingredient database was created as a Mongo Database and was connected to the application via mLab API. We also decided to use Firebase to store user authentication information. Firebase is optimized for maintaining user information such as user id, user name, email and password.

Skills used:
